# توثيق واجهات API — A4 POS Platform

يحتوي هذا الملف على دليل تفصيلي لكافة واجهات الـ API لنظام نقاط البيع A4، مع شرح تفصيلي لهيكل المجموعات والمسارات والقيود البرمجية للأمن والمخزون والورديات.

---

## 1. مجموعة Postman Collection
تم إنشاء ملف تعريف جاهز للاستيراد المباشر إلى برنامج Postman تحت مسار:
[A4_POS_API_Collection.json](file:///D:/a4.office/a4_agent_pack/docs/A4_POS_API_Collection.json)

### المتغيرات البيئية (Variables):
* `{{base_url}}`: المسار الأساسي لخدمات الـ API، القيمة الافتراضية: `http://localhost:5000/api`
* `{{token}}`: رمز التحقق JWT المرجع من عملية تسجيل الدخول (`accessToken`).
* `{{refresh_token}}`: رمز تجديد التحقق المرجع من عملية تسجيل الدخول.

## 2. المسارات والمجموعات (API Routes Map)

### أولاً: مجموعة الحسابات وصلاحيات الجلسة (Auth)
1. **تسجيل الدخول (POST `/auth/login`)**:
   * **الوصف**: تسجيل دخول الأدمن أو الكاشير وحفظ التوكن.
   * **المدخلات (JSON)**: `{"username": "admin", "password": "admin123"}`
2. **تجديد التوكن (POST `/auth/refresh`)**:
   * **الوصف**: تجديد جلسة التوكن المنتهية.
   * **المدخلات (JSON)**: `{"refreshToken": "token_string"}`
3. **تسجيل الخروج (POST `/auth/logout`)**:
   * **الوصف**: إنهاء الجلسة وتعطيل التوكن الحالي.
4. **ملفي الشخصي (GET `/auth/me`)**:
   * **الوصف**: الحصول على بيانات المستخدم وصلاحيات دوره الفعلي.

### ثانياً: إدارة الحسابات للمدير (Admin Users)
1. **عرض المستخدمين (GET `/admin/users`)**:
   * **الوصف**: عرض كافة حسابات الموظفين (أدمن فقط).
2. **إنشاء حساب موظف (POST `/admin/users`)**:
   * **الوصف**: إضافة كاشير أو أدمن جديد.
3. **تعديل بيانات الحساب (PATCH `/admin/users/:id`)**:
   * **الوصف**: تعديل اسم الكاشير أو رقم الهاتف.
4. **تغيير كلمة المرور (PATCH `/admin/users/:id/password`)**:
   * **الوصف**: تحديث كلمة المرور للحساب المحدد.
5. **تعطيل الحساب (PATCH `/admin/users/:id/disable`)**:
   * **الوصف**: إيقاف تفعيل حساب كاشير لمنعه من تسجيل الدخول.

### ثالثاً: إدارة التصنيفات والتسعير والأصناف (Catalog)
1. **التصنيفات (GET `/categories` / POST `/admin/categories` / PATCH `/admin/categories/:id`)**:
   * **الوصف**: إدارة وتعديل تصنيفات الكتب والأصناف الدراسية.
2. **فئات الأسعار (GET `/admin/price-tiers` / POST `/admin/price-tiers` / PATCH `/admin/price-tiers/:id`)**:
   * **الوصف**: تصنيف فئات البيع (مثل سعر قطاعي، جملة، سعر مدارس).
3. **المنتجات (GET `/products` / GET `/products/:id` / POST `/admin/products` / PATCH `/admin/products/:id`)**:
   * **الوصف**: إنشاء وتعديل بيانات الأصناف وتوزيع الأسعار حسب الفئات بدون إضافة صور للمنتجات.
4. **ملصقات الباركود (POST `/admin/products/:id/qr-labels`)**:
   * **الوصف**: إعداد ملصق الرمز التعريفي (QR) الخاص بالصنف لطباعته وتلصيقه على الكتاب، ولا يحتوي على أسعار أو كميات المخزون.

### رابعاً: إدارة المخزون والمخازن (Inventory)
1. **مؤشرات المخزون (GET `/admin/inventory`)**:
   * **الوصف**: عرض عدادات المخزون الكلي والتالف وتحت حد الأمان.
2. **شحن رصيد البداية (POST `/admin/inventory/stock-add`)**:
   * **الوصف**: إضافة كميات واردة وتعديل رصيد المخزن مباشرة.
3. **تسوية المخزون (POST `/admin/inventory/adjustment`)**:
   * **الوصف**: إضافة أو خصم كميات تالفة أو مفقودة مع وجوب إدخال السبب لتسجيله في سجل التدقيق.
4. **دفتر الحركات (GET `/admin/inventory/ledger`)**:
   * **الوصف**: دفتر القيد المزدوج للمخزون لعرض تواريخ الحركات وأسباب التغير بالتفصيل.

### خامساً: عمليات نقطة البيع والمرتجع (POS & Returns)
1. **مسح رمز المنتج (POST `/pos/scan-product`)**:
   * **الوصف**: إرسال الباركود المقروء بالماسح الضوئي للحصول على بيانات الصنف وسعره المناسب لإضافته للسلة.
2. **إتمام الفاتورة (POST `/pos/orders/checkout`)**:
   * **الوصف**: إتمام الدفع (يدعم الدفع الموزع كاش وفيزا وغيره) وتخفيض المخزون فورياً مع طباعة إيصال البيع الحراري.
3. **تفاصيل إيصال (GET `/pos/receipts/:id`)**:
   * **الوصف**: عرض هيكل الإيصال الحراري باللغة العربية.
4. **إعادة طباعة الإيصال (POST `/pos/receipts/:id/reprint`)**:
   * **الوصف**: إعادة طباعة إيصال مع قيد حركة AuditLog ورفع عداد مرات الطباعة.
5. **إرجاع الفاتورة (POST `/pos/orders/:id/return`)**:
   * **الوصف**: إرجاع كلي أو جزئي لكميات مباعة، وحساب نسبة الخصم بالتناسب، وإرجاع الكمية للمخازن، وإثبات حركة خروج نقدية (`PAY_OUT`) من الدرج للشيفت الحالي.

### سادساً: نظام الحجز المسبق ودفع العربون (Preorders)
1. **إنشاء حجز مسبق (POST `/pos/preorders`)**:
   * **الوصف**: إنشاء طلب حجز بالبيانات الإلزامية للعميل (الاسم ورقم الهاتف) ودفع عربون لا يقل عن نسبة الحد الأدنى المحددة للمنتج. لا يتم خصم المخزون في هذه المرحلة، ويتم توليد رمز استلام QR حراري.
2. **مسح رمز الاستلام (POST `/pos/preorders/scan`)**:
   * **الوصف**: مسح الرمز الحراري المطبوع للتأكد من حالة الطلب والكمية.
3. **تأكيد الاستلام والتسليم (POST `/pos/preorders/:id/pickup`)**:
   * **الوصف**: تسليم الحجز وتجميع المبلغ المتبقي، وخصم المخزون الفعلي، وإصدار إيصال بيع نهائي.
4. **عرض الحجوزات للأدمن (GET `/admin/preorders` / PATCH `/admin/preorders/:id/status`)**:
   * **الوصف**: لمتابعة الطلبات وتعديل حالاتها فور وصول توريدات جديدة للمخزن.

### سابعاً: الورديات وحركة الدرج (Shifts & Drawer)
1. **فتح وردية (POST `/shifts/open`)**:
   * **الوصف**: إدخال مبلغ عهدة البداية لبدء عمليات البيع اليومية.
2. **عرض الوردية (GET `/shifts/current`)**:
   * **الوصف**: متابعة الكاشير لمبيعاته الحالية ومبالغ النقدية المتوقع وجودها بالدرج.
3. **طلب الإغلاق (POST `/shifts/current/close-request`)**:
   * **الوصف**: إغلاق الشيفت وإدخال مبالغ العهدة الفعلية تمهيداً لمراجعة الإدارة.
4. **مراجعة واعتامد الورديات (GET `/admin/shifts` / POST `/admin/shifts/:id/approve` / `/reject`)**:
   * **الوصف**: فحص الفروقات بين مبالغ الدرج الفعلية ومبالغ النظام، واعتماد الوردية أو رفضها بملاحظة إدارية.

### ثامناً: مؤشرات الأداء والتقارير للأدمن (Reports & KPIs)
1. **مؤشرات الأداء (GET `/admin/kpis`)**:
   * **الوصف**: لوحة إحصائيات سريعة للرئيسية لحساب المبيعات، ومبالغ العربين، والورديات المعلقة، والمنتجات الأكثر طلباً.
2. **الفلاتر التفصيلية (GET `/admin/reports/sales` / `/preorders` / `/inventory` / `/shifts`)**:
   * **الوصف**: تقارير متكاملة تدعم الفلترة بنطاق التواريخ، الكاشير، التصنيف، رقم الوردية، حالة المخزون، أو حالات الحجز المسبق.
3. **تصدير التقارير (GET `/api/admin/reports/export`)**:
   * **الوصف**: تحميل ملف CSV مباشر داعم للغة العربية عبر ترميز UTF-8 BOM لفتحه وعرضه بشكل سليم داخل برنامج Microsoft Excel.

## 3. قيود عمل الـ APIs البرمجية الدائمة
* **عدم البيع أو الحجز بدون شيفت**: ترمي الواجهات خطأ `400` برمز `SHIFT_NOT_OPEN` في حال حاول الكاشير إجراء بيع، أو حجز، أو تسليم بدون فتح وردية نقدية نشطة لنفس الحساب.
* **منع تخفيض المخزون تحت الصفر**: يتحقق السيرفر دائماً من الكميات المتوفرة ولا يسمح نهائياً بتسجيل أي مبيعات أو تسوية تتجاوز الرصيد الفعلي للمنتج.
* **التوافق التام مع اللغة العربية**: كل رسائل الأخطاء، وحالات التأكيد والتحقق في الواجهات موجهة مباشرة لمستخدم النظام باللغة العربية الفصحى.
