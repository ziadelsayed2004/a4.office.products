# توثيق عمليات نسخ واسترجاع قاعدة البيانات — A4 Office Products POS

هذا المستند يشرح كيفية إدارة عمليات النسخ الاحتياطي (Backup) واسترجاع البيانات (Recovery) لقاعدة بيانات SQLite الخاصة بالمنصة.

---

## 1. عمليات النسخ الاحتياطي (Database Backup)

تم إعداد نظام نسخ احتياطي آلي وسهل الاستخدام يقوم بنسخ قاعدة البيانات بالكامل مع إلحاق طابع زمني دقيق بالتوقيت المحلي لمدينة القاهرة (`Africa/Cairo`).

### تشغيل النسخ الاحتياطي يدوياً
يمكنك تشغيل عملية النسخ الاحتياطي في أي وقت من المجلد الرئيسي للمشروع عبر تشغيل الأمر:
```bash
npm run db:backup
```

### آلية العمل ومكان الحفظ
- **مسار الحفظ**: يتم حفظ ملفات النسخ الاحتياطي في مجلد `backups/` في الجذر الرئيسي للمشروع.
- **تسمية الملفات**: تأخذ الملفات الصيغة التالية: `a4_pos_backup_YYYYMMDD_HHMMSS.db`.
- **التدوير الذاتي (Retention Policy)**: يحتفظ النظام تلقائياً بآخر **5 نسخ احتياطية فقط**، ويقوم بمسح النسخ الأقدم لتوفير المساحة وتجنب امتلاء القرص الصلب.

---

## 2. عمليات استرجاع البيانات (Database Recovery)

في حالة حدوث عطل أو الحاجة لاسترجاع نسخة احتياطية سابقة، يرجى اتباع الخطوات التالية بدقة:

### خطوات الاسترجاع:
1. **إيقاف خادم التطبيق (Backend Server)**:
   تأكد من إيقاف تشغيل السيرفر تماماً لتجنب وجود أي قفل (Lock) على قاعدة البيانات النشطة.
2. **تحديد ملف النسخة المطلوبة**:
   اذهب إلى مجلد `backups/` واختر ملف النسخة الاحتياطية التي ترغب في استرجاعها (مثال: `a4_pos_backup_20260710_222506.db`).
3. **استبدال قاعدة البيانات النشطة**:
   - قم بحذف ملف قاعدة البيانات النشط الموجود في `server/src/db/a4_pos.db` (أو خذ نسخة منه كاحتياط إضافي).
   - قم بنسخ ملف النسخة الاحتياطية المختارة وضعه في المسار `server/src/db/a4_pos.db` (أعد تسميته ليكون `a4_pos.db` تماماً).
4. **إعادة تشغيل الخادم**:
   قم بتشغيل السيرفر مرة أخرى. سيقوم التطبيق بفتح قاعدة البيانات المسترجعة وتفعيل قيود المفاتيح الأجنبية وفحص الجداول بشكل طبيعي.

---

## 3. تحسينات الأداء ومؤشرات البحث (Performance Indexes)

لضمان سرعة فائقة في معالجة العمليات الحسابية والبحث وتوليد الملصقات، تم إنشاء كشافات البحث (Indexes) التالية وتفعيلها برمجياً:
- `idx_inv_ledger_latest`: لتسريع عملية حساب الرصيد الحالي للمنتج من خلال جلب آخر حركة مخزنية مباشرة.
- `idx_orders_date` & `idx_orders_shift`: لتسريع تجميع بيانات الورديات وحساب إيرادات التقارير.
- `idx_preorders_shift` & `idx_preorders_status`: لتسريع عمليات البحث وفلترة الحجوزات المعلقة للعملاء.
- `idx_cash_movements_shift`: لتسريع التحقق المالي وتدقيق درج الكاشير أثناء الإغلاق.
